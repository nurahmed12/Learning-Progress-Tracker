ROUND(Cell or Value, number of digits) 
ROUND(E11, 0) Means non decimal round, Rounds to the nearest whole number, e.g., 10.3 > 10 and 10.7 > 11, if 10.5 > 11
ROUND(E11, 1 or any positive number) Means decimal round, Rounds the (1 or any positive number)th value to it's nearest whole number, e.g., ROUND(7.445, 1) > 7.4, ROUND(7.445) > 7.45
ROUND(E11, -1 or any negative number) Means tens, hundreds or more round, Rounds the (1 or any negative number) + 1th number from right most to it's nearest whole number based on the (1 or any negative number)th number, e.g., ROUND(123412, -1) > 123410, ROUND(123416, -1) > 123420, ROUND(123415, -2) > 123400
ROUNDUP(E11, 0 or Positive or Negative) Similar to ROUND() function but it only rounds to the next whole number, e.g., ROUNDUP(5.1, 0) > 6
ROUNDDOWN(E11, 0 or Positive or Negative) Opposite of ROUNDUP()
MROUND(E11, multiple) Rounds to the nearest Multiple, e.g., MROUND(12311, 4) > 12312 Means 12312 is the nearest to 12311 which can be devided by 4

- Editing Group (Sorting):

In order to generate random series you can use the formula =ROUND(Value1, (Value2 - Value1) * RAND(), 2) , this will generate random numbers between Value1 and Value2
1. RAND()
Returns a random decimal number between 0 and 1.
Example output: 0.2438, 0.8793, etc.
2. (b - a)
This is the range or width of the interval.
For example, if a = 5 and b = 10, then (b - a) = 5.
3. (b - a) * RAND()
Scales the random number into your desired range.
Now you get a random decimal between 0 and (b - a).
Example: If RAND() = 0.3, and (b - a) = 5, then this part is 1.5.
4. a + ...
This shifts the number up to start at a.
So you now get a number between a and b.
Example: a = 5, so 5 + 1.5 = 6.5
5. ROUND(..., 2)
Rounds the final result to 2 decimal places.

1. Formula Pattern
   =COUNTIF(<lookup_range>, <this_cell>) = 1
   ‚Ä¢ Returns TRUE if the value in <this_cell> appears exactly once in <lookup_range>.

2. Locking the Lookup Range
   ‚Ä¢ Fixed block (e.g. rows 10‚Äì17 in column‚ÄØK):
     =COUNTIF($K$10:$K$17, K10)=1
     ‚Äì $K$10:$K$17 stays the same when you copy down.
     ‚Äì K10 (no $) changes to K11, K12, etc.

   ‚Ä¢ Entire column:
     =COUNTIF($K:$K, K10)=1
     ‚Äì $K:$K always means the whole column‚ÄØK.
     ‚Äì Good for growing lists; slower on very large sheets; counts blanks too.

3. Criterion Cell Reference
   ‚Ä¢ Relative row (K10): adjusts to each row.
   ‚Ä¢ Absolute row ($K$10): always tests cell‚ÄØK10.

4. ‚Äú$‚Äù Cheat‚ÄëSheet
   ‚Ä¢ $ before column (e.g. $K) locks column.
   ‚Ä¢ $ before row (e.g. K$10) locks row.
   ‚Ä¢ $K$10 locks both column and row (single cell).
   ‚Ä¢ $K:$K locks entire column‚ÄØK.
   ‚Ä¢ $K$10:$K$17 locks that exact block.

CountIF function:

Quick Guide to COUNTIF in Excel

1. Function Structure  
   COUNTIF(range, criteria)  
   ‚Ä¢ Counts how many cells in‚ÄØrange meet the given criteria.

2. Parameters  
   ‚Ä¢ range: the block of cells to check (e.g. A1:A100, K:K)  
   ‚Ä¢ criteria: condition each cell is tested against (number, expression, text, wildcard, or cell reference)

3. Types of Criteria  
   ‚Ä¢ Number  
     ‚Äì COUNTIF(A1:A10, 5)  
       Counts cells exactly equal to 5.  
   ‚Ä¢ Expression (comparison)  
     ‚Äì COUNTIF(B1:B20, ‚Äú>100‚Äù)  
       Counts cells with values greater than 100.  
     ‚Äì Operators: =, <>, >, <, >=, <= (always in quotes)  
   ‚Ä¢ Text  
     ‚Äì COUNTIF(C1:C50, ‚ÄúApple‚Äù)  
       Counts cells exactly matching ‚ÄúApple‚Äù (case‚Äëinsensitive).  
   ‚Ä¢ Wildcards  
     ‚Äì `*` matches any sequence of characters  
       ‚Ä¢ COUNTIF(NameRange, "East*") finds ‚ÄúEast‚Äù, ‚ÄúEastern Campus‚Äù, etc.  
       ‚Ä¢ COUNTIF(NameRange, "*son") finds ‚ÄúJohnson‚Äù, ‚ÄúAnderson‚Äù, etc.  
     ‚Äì `?` matches exactly one character  
       ‚Ä¢ COUNTIF(CodeRange, "A?3") finds ‚ÄúAB3‚Äù, ‚ÄúA63‚Äù but not ‚ÄúA123‚Äù.  
       ‚Ä¢ COUNTIF(WordRange, "?at") finds ‚ÄúCat‚Äù, ‚ÄúBat‚Äù but not ‚ÄúThat‚Äù.  
   ‚Ä¢ Cell reference  
     ‚Äì COUNTIF(F1:F50, G1)  
       Counts cells equal to the value in G1.  
   ‚Ä¢ Expression with reference  
     ‚Äì COUNTIF(F1:F50, ‚Äú>‚Äù & G1)  
       Counts cells greater than the value in G1.  
   ‚Ä¢ Dates  
     ‚Äì COUNTIF(H1:H100, ‚Äú>‚Äù & DATE(2025,1,1))  
       Counts dates after Jan‚ÄØ1,‚ÄØ2025.  
     ‚Äì Can also reference a cell: ‚Äú>‚Äù & A1 where A1 holds a date.

4. Special Behaviors  
   ‚Ä¢ Blanks vs. non‚Äëblanks  
     ‚Äì COUNTIF(A1:A10, ‚Äú‚Äù) counts empty cells.  
     ‚Äì COUNTIF(A1:A10, ‚Äú<>‚Äù) counts cells that are not empty.  
   ‚Ä¢ Mixed data types  
     ‚Äì Numbers in quotes (e.g. ‚Äú5‚Äù) are treated as text, not numeric.  
   ‚Ä¢ Logical values and errors  
     ‚Äì COUNTIF will ignore TRUE, FALSE, and error values unless they match the criteria exactly.  
   ‚Ä¢ Whole‚Äëcolumn ranges (e.g. K:K)  
     ‚Äì Scans all 1,048,576 rows; includes blank cells in the count.  
     ‚Äì Use with care on very large sheets (performance).

5. Common Uses  
   ‚Ä¢ Count specific status  
     ‚Äì COUNTIF(StatusRange, ‚ÄúCompleted‚Äù)  
   ‚Ä¢ Count above/below thresholds  
     ‚Äì COUNTIF(ScoreRange, ‚Äú>=80‚Äù)  
     ‚Äì COUNTIF(ScoreRange, ‚Äú<50‚Äù)  
   ‚Ä¢ Partial text matching  
     ‚Äì COUNTIF(NameRange, ‚ÄúMc*‚Äù) finds names starting with ‚ÄúMc‚Äù.  
   ‚Ä¢ Compare to another cell  
     ‚Äì COUNTIF(DateRange, ‚Äú<‚Äù & DeadlineCell)  
   ‚Ä¢ Unique‚Äëvalue test  
     ‚Äì COUNTIF($K:$K, K10)=1 returns TRUE if K10 is unique in column‚ÄØK.

6. Locking References  
   ‚Ä¢ $A$1:$A$10 locks both column & row (never shifts)  
   ‚Ä¢ A$1 locks row only (column shifts when copying across)  
   ‚Ä¢ $A1 locks column only (row shifts when copying down)  
   ‚Ä¢ $K:$K locks entire column‚ÄØK  
   ‚Ä¢ $K$10:$K$17 locks block K10‚ÄìK17  

7. Relative vs. Absolute Ranges

   ‚Ä¢ For OR logic, combine multiple COUNTIFs or use SUMPRODUCT

   ‚Ä¢ Keep ranges tight (avoid entire columns) for better performance.

Quick Guide: Introduction to Pivot Tables in Excel

1. What Is a Pivot Table?  
   ‚Ä¢ An interactive summary tool that automatically aggregates, analyzes, and presents large data sets.  
   ‚Ä¢ Lets you ‚Äúpivot‚Äù (re‚Äëarrange) rows and columns to see different perspectives.

   ‚Ä¢ You have a detailed table of records (e.g., sales transactions, survey responses).  
   ‚Ä¢ You need quick totals, averages, counts, or subgroup analyses by category.  
   ‚Ä¢ You want an easy drag‚Äëand‚Äëdrop interface instead of complex formulas.

   ‚Ä¢ **Rows area**: Fields you want listed down the side (e.g., Salesman, District).

   ‚Ä¢ Right‚Äëclick a Value and choose **Value Field Settings** to switch from Sum to Count, Average, Min, Max, etc.

   ‚Ä¢ **Sales by month and district**: Rows = Month; Columns = District; Values = Sum of Sales TK.

   ‚Ä¢ Source data should have no blank header names and no completely blank rows/columns.

Logical Functions ‚Äì IF, AND, OR, NOT, IFS

1. IF  
   ‚Ä¢ Structure:  IF(logical_test, value_if_true, value_if_false)  
   ‚Ä¢ Tests one condition and returns one of two results.  
     ‚Äì Example:  =IF(A1>100, "High", "Low")  
       ‚Ä¢ If A1 > 100 returns ‚ÄúHigh,‚Äù otherwise ‚ÄúLow.‚Äù  
   ‚Ä¢ Nesting IFs (pre‚Äë2021):  
     ‚Äì =IF(A1>100, "High", IF(A1>50, "Medium", "Low"))  
     ‚Äì Can become complex with many levels‚Äîconsider IFS.

2. AND, OR, NOT  
   ‚Ä¢ AND(condition1, condition2, ‚Ä¶)  
     ‚Äì Returns TRUE only if ALL conditions are TRUE.  
     ‚Äì Example: =AND(A1>0, B1<10)  
   ‚Ä¢ OR(condition1, condition2, ‚Ä¶)  
     ‚Äì Returns TRUE if ANY condition is TRUE.  
     ‚Äì Example: =OR(A1="Yes", B1="Yes")  
   ‚Ä¢ NOT(condition)  
     ‚Äì Reverses a logical value: TRUE ‚Üí FALSE, FALSE ‚Üí TRUE.  
     ‚Äì Example: =NOT(C1="Complete")  (TRUE if C1 ‚â† ‚ÄúComplete‚Äù)

3. IFS (Excel‚ÄØ2016+ / 365)  
   ‚Ä¢ Structure:  IFS(test1, value1, test2, value2, ‚Ä¶, [TRUE, default])  
   ‚Ä¢ Evaluates tests in order and returns the first matching value.  
     ‚Äì Example: =IFS(A1>100, "High", A1>50, "Medium", A1>=0, "Low", TRUE, "N/A")  
   ‚Ä¢ No need to nest multiple IFs; cleaner and easier to read.

4. Combining with Other Functions  
   ‚Ä¢ IF + AND/OR for multi‚Äëcondition logic:  
     ‚Äì =IF(AND(A1>0, B1<5), "Valid", "Invalid")  
   ‚Ä¢ IFERROR to catch errors:  
     ‚Äì =IFERROR(1/0, "Error!") returns ‚ÄúError!‚Äù instead of #DIV/0!  
   ‚Ä¢ Use logical functions in array formulas for bulk logical tests (Office‚ÄØ365).

5. Common Pitfalls & Tips  
   ‚Ä¢ Blank vs. zero: ISBLANK(A1) or A1="" to test emptiness.  
   ‚Ä¢ Comparisons are case‚Äëinsensitive: "Yes" = "yes".  
   ‚Ä¢ Order matters in IFS‚Äîplace most specific tests first.  
   ‚Ä¢ Avoid deep nesting; switch to IFS or lookup tables (VLOOKUP / XLOOKUP) for complex branching.  
   ‚Ä¢ Always include a default catch‚Äëall (e.g. TRUE, "Other") as the last IFS pair.  

Lookup & Reference ‚Äì VLOOKUP, HLOOKUP, INDEX, MATCH, XLOOKUP

1. VLOOKUP  
   ‚Ä¢ Structure:  VLOOKUP(lookup_value, table_array, col_index, [range_lookup])  
   ‚Ä¢ lookup_value: value to find in first column of table_array  
   ‚Ä¢ table_array: block of cells (e.g. A1:D100)  
   ‚Ä¢ col_index: column number to return (1 = first column)  
   ‚Ä¢ range_lookup: FALSE for exact match, TRUE (or omitted) for approximate  
   ‚Ä¢ Example: =VLOOKUP(B2, Products!$A$2:$C$100, 3, FALSE)  
   ‚Ä¢ Pitfalls:  
     ‚Äì Only searches left‚Äëto‚Äëright  
     ‚Äì Requires first column sorted for approximate matches  
     ‚Äì Exact match must use FALSE  

2. HLOOKUP  
   ‚Ä¢ Structure:  HLOOKUP(lookup_value, table_array, row_index, [range_lookup])  
   ‚Ä¢ Same logic as VLOOKUP but searches in first row, returns from row_index  
   ‚Ä¢ Example: =HLOOKUP("Jan", Sales!$A$1:$M$5, 4, FALSE)  
   ‚Ä¢ Pitfalls:  
     ‚Äì Only top‚Äëto‚Äëbottom  
     ‚Äì Range must be sorted for TRUE  

3. INDEX  
   ‚Ä¢ Two forms:  
     ‚Äì Array form: INDEX(array, row_num, [column_num])  
     ‚Äì Reference form: INDEX(reference, row_num, [col_num], [area_num])  
   ‚Ä¢ Returns value at intersection of row_num & column_num  
   ‚Ä¢ Example: =INDEX(A2:C10, 5, 2) returns value in row‚ÄØ5, column‚ÄØ2 of that range  
   ‚Ä¢ Use with MATCH for flexible lookups  

4. MATCH  
   ‚Ä¢ Structure:  MATCH(lookup_value, lookup_array, [match_type])  
   ‚Ä¢ lookup_array: single row or column  
   ‚Ä¢ match_type: 0 for exact, 1 for <= (sorted ascending), -1 for >= (sorted descending)  
   ‚Ä¢ Returns position (number) of found item in lookup_array  
   ‚Ä¢ Example: =MATCH("Widget X", A2:A50, 0) returns the row offset  

5. INDEX + MATCH  
   ‚Ä¢ Combines strengths:  
     ‚Äì Lookup in any column/row, not limited to first  
     ‚Äì No need to sort for exact matches  
   ‚Ä¢ Example: =INDEX(C2:C50, MATCH(E2, A2:A50, 0))  
     ‚Äì Finds E2 in A2:A50, returns corresponding C column value  

6. XLOOKUP (Excel‚ÄØ365/2021+)  
   ‚Ä¢ Structure: XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])  
   ‚Ä¢ if_not_found: value when no match (default #N/A)  
   ‚Ä¢ match_mode: 0 exact, -1 exact or next smaller, 1 exact or next larger, 2 wildcard  
   ‚Ä¢ search_mode: 1 first‚Äëto‚Äëlast, -1 last‚Äëto‚Äëfirst  
   ‚Ä¢ Example: =XLOOKUP(E2, A2:A50, C2:C50, "Not found", 0, 1)  
   ‚Ä¢ Advantages:  
     ‚Äì Lookup left or right, vertical or horizontal  
     ‚Äì Built‚Äëin error handling  
     ‚Äì Supports wildcards and reverse search  

7. Common Tips & Pitfalls  
   ‚Ä¢ Always lock ranges with $ when copying formulas  
   ‚Ä¢ Use exact match (FALSE or 0) to avoid unexpected results
